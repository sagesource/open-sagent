# 方案：013-Web模块-前端风格优化方案

## 1. 背景与目的

### 1.1 背景

方案013（Web模块设计方案）已通过评审并进入实施阶段。当前方案中的前端UI设计偏向传统简约风格（基础配色、常规布局、无特效），与AI Agent产品的科技感定位不符。

### 1.2 目的

将前端UI风格从"简约基础"升级为"科技现代"，提升产品的视觉质感和用户体验，同时保持方案013的功能架构不变。

## 2. 设计方向

### 2.1 整体视觉风格

| 维度 | 当前方案 | 优化方向 |
|------|---------|---------|
| 色彩基调 | 白/灰底色 + 蓝色强调 | 深色科技底色 + 青蓝渐变强调 |
| 背景质感 | 纯色平铺 | 暗色网格/粒子背景 + 玻璃拟态卡片 |
| 边角处理 | 常规圆角(8-12px) | 更大圆角(16px) + 细微发光边框 |
| 动效反馈 | 无 | 平滑过渡 + 呼吸光效 + 打字机动画 |
| 字体排版 | 系统默认 | 更精致的字号层级 + 等宽字体用于代码 |

### 2.3 全局背景特效

在 `App.tsx` 根层级增加科技风格背景组件：

1. **暗色网格背景**：使用CSS `background-image` 绘制细微网格线，营造科技感
2. **顶部光晕**：页面顶部中央添加一个大的弥散光晕（使用radial-gradient），颜色为 `--accent-primary` 的低透明度版本
3. **可选粒子效果**：使用轻量CSS动画实现缓慢漂浮的光点（不引入额外库）

### 2.4 玻璃拟态（Glassmorphism）

关键面板和卡片采用玻璃拟态效果：

```css
.glass-panel {
  background: var(--bg-glass);
  backdrop-filter: blur(12px);
  -webkit-backdrop-filter: blur(12px);
  border: 1px solid var(--border-color);
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.3);
}
```

应用位置：侧边栏、消息气泡、输入框区域、登录卡片。

### 2.6 动效规范

```css
/* 通用过渡 */
--transition-fast: 0.15s ease;
--transition-normal: 0.3s ease;
--transition-slow: 0.5s ease;

/* 消息入场动画 */
@keyframes messageIn {
  from { opacity: 0; transform: translateY(10px); }
  to { opacity: 1; transform: translateY(0); }
}

/* 脉冲呼吸（用于AGENT_ACTION） */
@keyframes pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.5; }
}

/* 闪烁光标 */
@keyframes blink {
  0%, 100% { opacity: 1; }
  50% { opacity: 0; }
}

/* 发光边框呼吸 */
@keyframes glow {
  0%, 100% { box-shadow: 0 0 5px var(--border-glow); }
  50% { box-shadow: 0 0 15px var(--border-glow); }
}
```

### 2.8 技术栈调整

当前方案前端依赖不变，不新增UI库（保持轻量）。所有效果使用纯CSS实现：

- `backdrop-filter` 实现玻璃拟态
- CSS `@keyframes` 实现所有动画
- `linear-gradient` 实现渐变
- 现有 `react-markdown` + `react-syntax-highlighter` 继续用于内容渲染

> 注：如后续需要更复杂动效（如3D、粒子系统），可评估引入 `framer-motion` 或 `canvas` 方案，本次优化保持零新增依赖。
