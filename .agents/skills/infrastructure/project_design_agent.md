## Prompt模块设计方案

Prompt是Agent运行的关键部分，本模块主要是实现支持自定义&占位符替换的System Prompt的加载

Prompt支持从工程目录或者指定路径加载Prompt模板，然后对模板内的占位符{{占位符:默认值}}进行替换，使用方可以基于预置模板生成SYSTEM PROMPT

优先级：指定路径 > 工程目录

在core模块定义与prompt相关的核心模型与功能，在infrastructure模块完成具体实现（指定路径加载和指定工程目录加载）

## Memory模块设计方案

Memory是Agent运行的关键部分，提供了添加对话历史、记忆压缩、为Agent调用大模型提供历史对话上下等能力。目的在于控制上下文规模，减少模型幻觉，提高模型处理准确度的能力

核心属性：

- 对话历史（完整的历史对话信息列表）
- 记忆历史（所有压缩后的记忆列表）
- 未压缩对话历史（每次压缩对话时，使用最新的记忆历史 + 未压缩对话历史 进行记忆压缩，压缩完成后需要清空）

core模块Memory模型，定义了其基本的功能：添加对话历史、压缩&保存记忆

无论何种实现，在记忆压缩阶段必须实现如下能力：

- 使用最新的记忆历史 + 未压缩对话历史 进行记忆压缩
- 在压缩的记忆中，可以搜寻到本次压缩的记忆中包含的最后一条对话ID和最后一条记忆历史ID，目的在于可以关联到一条记忆所对应的对话历史和记忆历史

在infrastructure模块完成具体Memory的实现：

1. SimpleMemory: 

- 将某一个对话窗的对话历史保存在内存中；
- 使用滑动时间窗模式记忆压缩(即每次执行压缩记忆时，在未压缩对话历史中保存最近50条会话信息)；50条可配置
- 压缩时剔除SYSTEM消息；如果待压缩的消息中有TOOL消息，则只保留最新的一次

## Agent模块设计方案

Agent是Sagent的核心组件。在core中定义Agent统一模型，即用户传入消息(文本、文件或文本+多文件)，Agent调用大模型完成补全思考并返回最终结果

Agent支持同步、异步流式响应结果（支持中断）；同步、异步响应结果 - 调用不同的Completion方法完成该过程

Agent核心属性：工具注册表(可选, 如果Agent配置参数选择开启则必须传递)；Memory(非必需，比如只用来生成标题或做记忆压缩的Agent，无需记忆)；Completion；Prompt（初始化Agent实例是要完成Prompt的构建）

Agent核心配置：是否开启工具支持；循环次数(可选，如果使用ReActAgent模式则配置参数必传)；温度

Agent核心方法：同步调用、同步流式调用、异步调用、异步流式调用

在infrastructure模块完成具体的Agent实现

1. SimpleAgent

- 无循环执行
- 每次执行前调用记忆执行记忆压缩
- 每次调用Completion时，组装消息历史逻辑：SYSTEM PROMPT + 记忆中未压缩的对话历史 + 最新的对话
- Completion响应完成后，要在记忆中保存对话历史(本次用户消息 + 本次模型响应消息)
- 如果开启工具支持，则调用Completion时要传递工具列表；如果响应了TOOL消息，要执行工具，并将执行结果作为TOOL消息保存在记忆中
