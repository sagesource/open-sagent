# Open Sagent

一个践行Harness模式，基于AI开发的AI AGENT工程

善用AI，不被AI所控

# 开发流程&思路

1. 新建AGENTS.md文件

> 在 AGENTS.md 中，用极简的语言描述架构的核心原则，并告诉 AI 代理去哪里寻找详细的实现方案。
> 
> 初始化项目脚手架之后，第一步应该是创建一个精简的 AGENTS.md。随着项目的演进和特定工作流的固化，再逐步将复杂的、特定场景的操作沉淀为 .agents/skills/ 目录下的各个 Skill

- 确立基线规则（Baseline Rules）： 告诉AI我的技术选型和偏好
- 作为所有工具的通用锚点（Cross-tool Anchor）：无论后续切换到哪个AI工具，项目核心的上下文都可以无缝迁移
